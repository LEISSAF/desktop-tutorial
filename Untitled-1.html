    • COMUNICAÇÃO CLIENT X SERVER

    (Voltar)

    A comunicação cliente-servidor é um modelo de arquitetura de software que divide as tarefas entre dois componentes principais: o cliente e o servidor. O cliente é o componente que solicita serviços do servidor, enquanto o servidor é o componente que fornece esses serviços.

    Comunicação cliente-servidor é uma forma eficiente de distribuir recursos e tarefas. O cliente pode se concentrar na interface com o usuário, enquanto o servidor pode se concentrar no processamento de dados e no armazenamento de informações.

    A comunicação cliente-servidor é utilizada em uma ampla variedade de aplicações, incluindo:

    Web: Quando você acessa um site, seu navegador (cliente) envia uma solicitação para o servidor web que hospeda o site. O servidor web retorna a página HTML solicitada, que é exibida no navegador.

    Email: Quando você envia um e-mail, seu cliente de e-mail (cliente) envia a mensagem para o servidor de e-mail. O servidor de e-mail recebe a mensagem e a encaminha para o servidor de e-mail do destinatário. O servidor de e-mail do destinatário entrega a mensagem ao cliente de e-mail do destinatário.

    Bancos de dados: Os clientes de banco de dados (clientes) enviam solicitações ao servidor de banco de dados (servidor). O servidor de banco de dados responde às solicitações, fornecendo os dados solicitados.

    A comunicação cliente-servidor é baseada em trocas de mensagens. O cliente envia uma mensagem ao servidor, que responde com outra mensagem. A mensagem do cliente pode conter uma solicitação de serviço, dados a serem processados ou ambos. A mensagem do servidor pode conter a resposta à solicitação, os resultados do processamento ou ambos.

    A comunicação cliente-servidor pode ser síncrona ou assíncrona. Na comunicação síncrona, o cliente espera uma resposta do servidor antes de continuar. Na comunicação assíncrona, o cliente envia uma mensagem ao servidor e continua a executar outras tarefas. O servidor envia uma resposta ao cliente quando estiver pronto.

    A comunicação cliente-servidor é um modelo de arquitetura eficiente e versátil que é utilizado em uma ampla variedade de aplicações.

    SURGIMENTO DA INTERNET

    (Voltar)

    A internet surgiu na década de 1960, nos Estados Unidos, como uma rede de computadores para fins militares. O objetivo era criar um sistema de comunicação descentralizado, que pudesse resistir a ataques nucleares. O projeto foi liderado pela Agência de Projetos de Pesquisa Avançada (ARPA), do Departamento de Defesa dos Estados Unidos. Em 1969, foi estabelecida a primeira conexão entre dois computadores, na Universidade da Califórnia em Los Angeles (UCLA) e o Instituto de Pesquisa de Stanford. A rede, inicialmente chamada de ARPANET, foi expandindo-se gradualmente para outras instituições de pesquisa e universidades. Em 1974, foi criado o protocolo TCP/IP, que padronizou a comunicação entre os computadores da rede.

    Início da década de 1980, a ARPANET foi aberta ao público, e a internet começou a se popularizar. Em 1985, já havia cerca de 1 milhão de computadores conectados à rede. A internet continuou a crescer rapidamente nas décadas seguintes. Em 1990, havia cerca de 10 milhões de computadores conectados, e em 2000, já havia mais de 1 bilhão.

    LINGUAGENS DE PROGRAMAÇÃO

    (Voltar)

    Linguagens de programação em resumo

    Linguagens de programação são conjuntos de regras que definem como os computadores devem entender e executar instruções. São usadas para criar software, que é o conjunto de instruções que controlam o comportamento de um computador. Existem muitos tipos diferentes de linguagens de programação, cada uma com suas próprias características e aplicações.

    HTML

    (Voltar)

    O que é HTML?

    Linguagem de Marcação de Hipertexto (HTML) é uma linguagem de computador que compõe a maior parte das páginas da internet e dos aplicativos online. Um hipertexto é um texto usado para fazer referência a outros textos, enquanto uma linguagem de marcação é composta por uma série de marcações que dizem para os servidores da web qual é o estilo e a estrutura de um documento. O HTML não é considerado uma linguagem de programação, já que ele não pode criar funcionalidades dinâmicas. Ao invés disso, com o HTML, os usuários podem criar e estruturar seções, parágrafos e links usando elementos, tags e atributos. Confira abaixo alguns dos usos mais comuns para o HTML: Desenvolvimento web. Os desenvolvedores usam códigos HTML para projetar como um navegador vai exibir os elementos das páginas, como textos, hiperlinks e arquivos de mídia. Navegação na internet. Os usuários podem navegar facilmente e inserir links entre páginas e sites relacionados, já que o HTML é amplamente usado para incorporar hiperlinks.

    TAGS

    (Voltar)

    O que são tags HTML?

    Tags são usadas para informar ao navegador a estrutura do site. Ou seja: quando se escreve um código em HTML, as tags serão interpretadas pelo navegador, produzindo assim a estrutura e o conteúdo visual da página. A principal característica das tags é estarem sempre dentro dos sinais de chevron (sinal de “maior que” e “menor que”), ou seja: < >. As tags HTML são divididas em dois tipos: as que precisam de fechamento e as que não precisam de fechamento. As tags que precisam de fechamento possuem a sintaxe < > , já as que não precisam de fechamento possuem como estrutura < />. Além disso, uma mesma tag pode receber um ou mais atributos, que possuirá um valor que modifica sua estrutura ou funcionalidade.

    LINKS

    (Voltar)

    o que são links html ?

    Links HTML são elementos de código que permitem aos usuários navegar de uma página da web para outra. Eles são usados para criar hiperlinks, que são palavras ou imagens que, quando clicadas, abrem uma nova página da web.

    ANCORAGEM

    (Voltar)

    Ancoragem no HTML é o uso da tag para criar um link para uma parte específica de uma página web. Isso pode ser usado para criar atalhos para pontos específicos do conteúdo, como uma seção, um parágrafo ou até mesmo uma imagem. Para criar uma âncora, você precisa usar o atributo name na tag . O valor do atributo name é o nome da âncora.
